# Structure And Content Parity

Before adding or changing a skeleton, inspect the final component and its shared layout, tokens, and data states. Build the skeleton from the same semantic layout primitives or a shared loading variant. Match headers, toolbars, cards, tables, rows, avatars, media aspect ratios, and responsive breakpoints.

When the page structure changes, the corresponding skeleton is part of the same change. A stale skeleton that flashes an obsolete layout is a regression even if the final state is correct.
