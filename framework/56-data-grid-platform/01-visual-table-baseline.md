# Visual Table Baseline

The existing project table is the first visual reference. Preserve its governed theme tokens, header hierarchy, row density, column alignment, numeric formatting, status treatments, sticky regions, footer/aggregate treatment, RTL behavior, responsive overflow, hover, focus, and selected states.

A table baseline must define the table shell, toolbar, search/filter/sort controls, column menu, row states, cell states, empty/error/loading states, pagination or cursor controls, aggregate footer, selection affordance, and keyboard behavior. Do not copy CSS into a new page; expose shared variants and semantic tokens.
