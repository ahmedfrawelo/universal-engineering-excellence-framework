# Application Shell Design Checklist

- [ ] Shell root, sidebar, header, route outlet, overlays, notifications, theme, and loading boundaries are mapped.
- [ ] Navigation IDs, routes, permissions, active descendants, badges, collapse state, deep links, and back/forward behavior are defined.
- [ ] Header/page chrome hierarchy, sticky behavior, actions, identity, search, notifications, and z-index ownership are defined.
- [ ] Page regions, route metadata, transition cancellation, authorization loading, and failure recovery are verified.
- [ ] Motion vocabulary, timing, interruption, reduced motion, and rapid navigation behavior are tested.
- [ ] Responsive, RTL, keyboard, touch, screen reader, zoom, text scaling, focus return, and long-label states pass.
- [ ] Shell skeleton, route loading, stale/offline/retry/fatal states, and realtime badge reconciliation are defined.
- [ ] Visual regression and performance budgets cover themes, viewport modes, repeated navigation, and no duplicate listeners.
