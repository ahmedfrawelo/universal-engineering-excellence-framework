# Security Hardening Checklist

Version: 1.1.0

Use this checklist with repository evidence. Marking an item complete requires a command, test, trace, screenshot, review record, or direct implementation reference.

- [ ] Assets, actors, boundaries, sensitive data, and abuse cases recorded
- [ ] Authentication, per-action authorization, tenant isolation, validation, and secrets reviewed
- [ ] CSRF, XSS, injection, SSRF, rate limits, idempotency, and safe errors evaluated
- [ ] Dependencies, configuration, logs, audit events, and recovery checked
- [ ] Automated scans and adversarial tests recorded with findings owners
