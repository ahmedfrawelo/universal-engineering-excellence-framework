# Visual Composition Checklist

- [ ] The primary task and action hierarchy are clear in the first viewport.
- [ ] Shell, content width, columns, and empty regions were inspected at desktop, tablet, and mobile sizes.
- [ ] Sparse and dense content layouts avoid dead space and accidental half-page composition.
- [ ] Forms use intentional grouping, field order, widths, labels, validation placement, and submit affordance.
- [ ] Typography, contrast, spacing, alignment, and section rhythm create visible hierarchy.
- [ ] Loading, empty, error, validation, long-label, RTL, theme, reduced-motion, and zoom states were checked.
- [ ] Visual evidence includes screenshots or live inspection, not only build/test output.
- [ ] No page-specific workaround bypasses the shared shell, tokens, components, or responsive contracts.
