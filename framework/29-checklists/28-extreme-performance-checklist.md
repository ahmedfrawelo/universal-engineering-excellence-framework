# Extreme Performance Checklist

Version: 1.1.0

Use this checklist with repository evidence. Marking an item complete requires a command, test, trace, screenshot, review record, or direct implementation reference.

- [ ] Budgets and representative measurement environment recorded
- [ ] Rendering, bundle, network, API, database, memory, CPU, and cost hotspots measured
- [ ] Duplicate work, N+1 paths, unnecessary renders, large lists, and inefficient queries addressed
- [ ] Caching, batching, parallelism, and indexes have correctness and invalidation evidence
- [ ] Before and after percentiles plus regression thresholds recorded
