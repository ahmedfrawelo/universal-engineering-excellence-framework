# Design Intelligence Checklist

- [ ] Project source of truth and existing design docs were identified.
- [ ] Font files, stacks, language coverage, roles, weights, sizes, and line heights were extracted.
- [ ] Colors and semantic theme mappings were extracted for light, dark, and system modes.
- [ ] Icon library, family style, stroke weight, sizes, labels, and hit targets were extracted.
- [ ] Outline, border, radius, shadow, spacing, motion, and breakpoint tokens were extracted.
- [ ] Each value is classified as extracted, inferred, or proposed.
- [ ] Missing roles have recommendations with accessibility, bundle, owner, and migration evidence.
- [ ] No new library or page-local token bypasses the existing source of truth.
