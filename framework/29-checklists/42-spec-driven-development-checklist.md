# Spec-Driven Development Checklist

- [ ] Spec-driven applicability was checked.
- [ ] Specification or equivalent requirement artifact exists when needed.
- [ ] What/why is separated from how.
- [ ] Non-goals are explicit.
- [ ] Acceptance criteria are measurable.
- [ ] Ambiguities are resolved or documented.
- [ ] Technical plan traces decisions to requirements.
- [ ] Security, performance, accessibility, data, and operational constraints are covered.
- [ ] Tasks have owners, dependencies, validation, and done criteria.
- [ ] Parallel tasks have disjoint write sets.
- [ ] Convergence check was completed before final claims.
