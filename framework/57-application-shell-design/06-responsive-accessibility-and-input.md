# Responsive, Accessibility, and Input Contract

- Define shell modes by available width and content needs, not device labels alone.
- Support mouse, touch, keyboard, screen reader, zoom, text scaling, high contrast, RTL, portrait/landscape, and safe-area insets.
- Maintain visible focus, logical tab order, landmarks, current-page semantics, expanded/collapsed announcements, and focus return after menus/drawers.
- Avoid hover-only discovery, clipped labels, inaccessible icon-only actions, and horizontal overflow that hides required navigation.
- Test long translated labels, large text, empty navigation, permission changes, offline mode, and narrow viewports.
