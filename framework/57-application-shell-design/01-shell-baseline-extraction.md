# Shell Baseline Extraction

Before changing the shell, produce an evidence map rather than relying on a screenshot alone.

- Capture the shell at representative viewport sizes and in light/dark/system themes where supported.
- Inventory logo/brand area, primary and nested navigation, active/hover/focus/disabled states, collapse behavior, badges, user menu, search, notifications, breadcrumbs, page title/actions, help/chat, and footer utilities.
- Extract design tokens for dimensions, spacing, typography, colors, borders, elevation, icon size/stroke, z-index, and motion timing/easing.
- Map route-to-navigation relationships, permission visibility, external links, deep links, browser back/forward, and restored shell state.
- Record intentional exceptions and unresolved drift with an owner and verification path.
