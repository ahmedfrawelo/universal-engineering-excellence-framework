# Header and Page Chrome Contract

- Define the hierarchy and ownership of brand, global search, breadcrumbs, page title, contextual actions, notifications, identity, help, and theme controls.
- Keep global actions visually distinct from page actions and permission-gated actions.
- Define sticky behavior, scroll thresholds, compact mode, overlap rules, safe-area insets, and z-index ownership.
- Page headers must expose loading, stale, error, permission, and unsaved-change states without shifting the main content unexpectedly.
- Tooltips, menus, dialogs, and command palettes follow the shared overlay and focus contract.
