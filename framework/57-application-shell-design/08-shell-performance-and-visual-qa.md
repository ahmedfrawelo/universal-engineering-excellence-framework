# Shell Performance and Visual QA

- Measure first meaningful shell paint, interactive shell time, route transition time, long tasks, layout shift, memory, and animation frame stability.
- Lazy-load feature navigation and heavy menus without delaying core identity and primary navigation.
- Verify no duplicate subscriptions, listeners, overlays, timers, or route components after repeated navigation.
- Run visual regression at key viewport/theme/RTL/reduced-motion states and inspect screenshots for clipping, overlap, contrast, focus, and incorrect active routes.
- Keep shell performance budgets release-blocking because every page inherits its cost.
