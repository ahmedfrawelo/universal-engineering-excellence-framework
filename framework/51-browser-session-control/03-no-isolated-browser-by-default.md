# No Isolated Browser by Default

An isolated browser is prohibited for every task governed by an explicit or established Chrome surface.

- Do not launch a new browser or context to bypass missing access.
- Do not use a fresh context to claim the user's work is complete.
- User approval does not convert an isolated context into the user's Chrome session. If the user explicitly requests a separate isolated test, treat it as a different task and never use it to satisfy or verify the Chrome task.
- A Codex-titled connector window, newly launched automation Chrome, temporary profile, or unrecognized profile is unusable and must be blocked. A control indicator on a proven existing extension-claimed user tab requires provenance verification, not automatic rejection.
