# Third-Party Attribution

## Purpose

External specification templates, prompts, code, docs, and packaged workflows must be handled with license-aware discipline.

## Rules

- Prefer adapting ideas over copying external text, templates, or code.
- If substantial external material is copied or modified, preserve the required copyright and permission notice.
- Record the source URL, reviewed commit or version, license, copied/adapted boundary, and files affected.
- Do not import third-party templates into UEEF without a third-party attribution file and validation coverage.
- If license terms are unclear, do not incorporate the material. Link to it as an external reference instead.

## Required Attribution Fields

- Source name.
- Repository or documentation URL.
- Reviewed commit, tag, or access date.
- License and copyright holder.
- What was reused.
- What was not copied.
- Integration boundary and validation evidence.
