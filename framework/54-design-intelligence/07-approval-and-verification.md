# Approval And Verification

Every design recommendation must link to evidence, affected consumers, and a verification plan. Validate theme parity, accessibility contrast, responsive behavior, icon alignment, font loading, visual regression, bundle impact, and ownership. A generated report is evidence of discovery, not approval to change the product.

The recommendation analyzer must be treated as a review queue. P1 findings require human or accountable-owner review before implementation; P2 findings require evidence-backed design-system work; P3 findings may be maintained with periodic review.
