# Color And Theme

Extract semantic color roles and their light/dark/system mappings. Review contrast for text, icons, borders, focus, disabled, hover, selected, error, warning, success, and data visualization states. Prefer OKLCH or the project's existing color representation; do not replace a coherent project palette merely to follow a generic trend.

If a role is missing, add it to the central theme source and map it in every supported theme before consuming it.
