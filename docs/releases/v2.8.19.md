# UEEF 2.8.19

Spec-driven development release inspired by GitHub Spec Kit.

## Added

- Added `framework/60-spec-driven-development/` for constitution, specification, clarification, technical-plan translation, task breakdown, consistency analysis, convergence, extension/preset/bundle governance, and third-party attribution.
- Added `framework/27-quality-gates/33-spec-driven-development-gate.md`.
- Added `framework/29-checklists/42-spec-driven-development-checklist.md`.
- Added `framework/38-templates/29-spec-driven-development-template.md`.
- Added `docs/third-party/spec-kit-attribution.md` with source, commit, and license attribution.

## Changed

- Runtime loader, core, master loader, runtime sequence, and generated Codex AGENTS now include spec-driven development rules.
- Quality gate selector now routes specification-driven, Spec Kit, requirements, acceptance criteria, ambiguity, and convergence tasks through pack 60.
- Validators now require the pack, gate, checklist, template, attribution, release note, and selector coverage.
