# UEEF 2.17.1

Release date: 2026-07-23

Codex-safety hardening so an installed runtime stays fast, capable, and non-hostile on this machine:

- Resolve `CODEX_HOME` through `scripts/resolve-codex-home.ps1` with machine default `E:\shared folder\codex-home` when unset.
- Make Open Design skill installation opt-in via `sync-runtime.ps1 -InstallOpenDesignSkills` (no default Python/GitHub fetch on every sync).
- Route with `preferredModel = inherit` and proportional reasoning: T0 = low, T1/T2 = medium, T3/T4 = high. Do not recommend fictional `gpt-5.6-*` model IDs.
- Replace the recursive workspace walk in `environment-bootstrap.ps1` with a shallow scan that never traverses `node_modules`, `.git`, `dist`, `.next`, `.venv`, `target`, `build`, `coverage`, `.turbo`, or `vendor`.
- Keep generated `AGENTS.md` compact and Codex-safe: prefer the installed Chrome control plugin against the user's existing tabs/profile; forbid Playwright, chrome-devtools, in-app browser, IDE Simple Browser, second profiles, and new contexts as substitutes.
- Soften loader refresh to once per task (or when runtime/version/policy may have changed), not every trivial follow-up.
- Allow Codex Chrome plugin or Claude `mcp__node_repl__js` bridge in browser preflight; still hard-fail alternate browsers.
