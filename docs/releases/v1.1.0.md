# UEEF v1.1.0

UEEF 1.1.0 is a backward-compatible expansion that turns theme, responsive layout, interaction behavior, identity models, security, and performance into one governed product system.

## Highlights

- Existing-theme compatibility plus centralized light, dark, and system architecture and persistence.
- Semantic tokens, controlled soft corners, and a solid-line UI policy.
- Responsive-first contracts for pages, forms, tables, dashboards, media, overlays, orientation, zoom, and short heights.
- Unified dropdown, menu, popover, drawer, panel, dialog, and tooltip behavior.
- Deterministic outside click, same-trigger toggle, peer policy, Escape, focus restoration, scroll locking, collision, portals, and layers.
- Frontend, backend/API, database, and tenant security hardening with explicit tests.
- Rendering, API, query, network, asset, and bundle performance standards with budgets.
- Five decision graphs, one release gate, one scorecard, six checklists, and five templates.
- Application models for SaaS, internal employee, public free, and hybrid products plus design-system reuse governance.

## Migration and Upgrade

Pull the release, run validation, and re-run the relevant installer. Existing projects retain their identity: extend current tokens and components before replacing them. New frontend projects establish semantic themes and responsive architecture before broad page work. Migrate overlays incrementally through the shared interaction coordinator with regression tests.

```powershell
git pull
powershell -ExecutionPolicy Bypass -File .\scripts\validate-framework.ps1
powershell -ExecutionPolicy Bypass -File .\scripts\install-codex.ps1
```

Use `scripts/update.ps1` for updates and the installer backup for rollback. Verify with `scripts/ueef-status.ps1` and `scripts/check-runtime-drift.ps1`.

The compact runtime contract remains `Loaded: boot-loader, core-system`; packs 45-47 are selected modules.
