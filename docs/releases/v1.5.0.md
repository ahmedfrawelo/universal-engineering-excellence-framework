# UEEF v1.5.0

UEEF 1.5.0 adds Browser Session Control for user-owned browser workflows.

## Changes

- Browser tasks use the browser and tab the user actually opened.
- Authenticated workflows require visible domain and signed-in-state verification.
- Cookies, passwords, tokens, local storage, and browser profile stores are never inspected.
- New or isolated browsers, contexts, and profiles are prohibited by default and require explicit approval.
- Missing user browser access blocks the task with a clear request for the user to open the target tab and sign in.
- Added browser session preflight, quality gate, checklist, loader selection, and validation.
