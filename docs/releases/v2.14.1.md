# UEEF 2.14.1

Release date: 2026-07-23

## User-owned browser hardening

Browser work that depends on the user's session, visual verification, or an authenticated site must stay on the existing user-owned Chrome window and tab. The runtime now explicitly prohibits second browsers, profiles, contexts, Cursor/IDE Simple Browser, in-app browser tools, `browser_*`, `browser.newContext`, and `browser.launch` as fallback surfaces.

An isolated/local browser test requires an explicit separate user request. It must not access, perform, satisfy, or verify the user-owned Chrome task. If readiness and same-tab failover cannot establish the existing tab, the agent must stop rather than open a replacement browser.
