# UEEF 2.8.17

Project-context and skill-asset hardening release.

## Added

- Added `assets/ueef-skill-icon.svg` as the official UEEF skill/display icon asset for Codex and related skill surfaces.
- Added `assets/ueef-display.json` as the display metadata contract that references the icon.
- Added `scripts/project-context-map.ps1` and `scripts/project-context-map.sh` to produce a bounded project map before broad implementation work.
- The context map reports manifests, shared owners, feature/module folders, design-system candidates, tests, and generated/output folders.
- Added runtime drift reporting in `ueef-status.ps1` so stale runtimes cannot report unqualified `ACTIVE`.
- Added motion, animation, transition, and easing routing through UIUX plus `emil-design-eng`.

## Changed

- Large-project reuse now requires running the context map or an equivalent repository map before broad or unfamiliar project work.
- Runtime sync now advertises the skill icon asset, display metadata, and project-context-map helper in the generated loader and Codex AGENTS instructions.
- Validation now checks the asset, scripts, documentation, and script execution.

## Verification

- Run `scripts/project-context-map.ps1 -Path . -MaxItems 10` on Windows.
- Run `sh scripts/project-context-map.sh . 10` on Unix.
- Run `scripts/validate-framework.ps1` and `scripts/validate-framework.sh`.
