# UEEF v2.8.15

Release date: 2026-07-15

## Summary

This release strengthens the default engineering contract for UI skill routing, file organization, file size, standalone files, response quality, backend performance, and SSR evaluation.

## Changes

- Requires UI/UX work to apply `ui-ux-pro-max` and `impeccable` together when available.
- Requires every new file to live under an owned feature, layer, package, route, docs, tests, scripts, generated-artifact, deployment, or configuration folder.
- Prohibits root-level file dumps, generic mixed folders, and hidden standalone-file systems unless they are standard entrypoints, documented configuration, owned one-off scripts, or explicit requested artifacts.
- Requires large mixed-responsibility files to be split by ownership before completion, or documented with a concrete reason.
- Requires backend work to evaluate server-side pagination, filtering, sorting, aggregation, projection, caching and invalidation, cancellation, concurrency, serialization, authorization cost, rate limits, and burst behavior where applicable.
- Requires frontend route work to evaluate SSR, SSG, streaming, route-level pre-rendering, or server components when SEO, public content, first paint, slow client boot, or first-view data volume makes server rendering useful.
- Requires frontend work to prevent over-rendering through state ownership, selectors, subscriptions, effects, memoization, virtualization, computed values, and component boundaries.
- Requires backend work to prevent over-render and over-refresh triggers such as over-fetching, over-serialization, repeated queries, noisy realtime broadcasts, broad cache invalidation, and unbounded recomputation.
- Requires animations to be smooth, interruptible, reduced-motion aware, and compositor-friendly, using transform and opacity by default and avoiding layout thrash or unrelated re-renders.
- Requires task-scope discipline so agents fix only the requested work, direct blockers, and current-change regressions, while recording unrelated pre-existing errors separately.
- Strengthens final-response quality rules so completion and verification claims need current evidence.
