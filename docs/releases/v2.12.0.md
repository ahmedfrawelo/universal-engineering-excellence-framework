# UEEF 2.12.0

Release date: 2026-07-23

## Highlights

- Introduces an optional, project-local `.ueef/specs/<id>` workflow with a safe generator and `Draft`/`Ready` validation modes. It makes the specification, technical plan, task evidence, and convergence record executable without forcing durable artifacts for small tasks.
- Introduces a registry-backed capability doctor for skills, plugins, and MCP configuration. It reports declared, configured, installed, enabled, and callable states separately; it never starts configured MCP commands, makes network requests, or prints configured command arguments/endpoints.
- Introduces proportional capability profiles: core-only for self-contained requests; selective skills/MCPs for directly relevant work; assured verification for high-impact work.
- Includes the capability registry in the allowlisted runtime payload and keeps the tracked-file-only release policy intact.

## Validation

- Spec workflow, capability health, and capability profile tests pass.
- Windows and Unix framework validators pass.
- Environment bootstrap tests pass from outside the repository working directory.
- Runtime hardening remains a release gate after the source changes are committed and therefore eligible for the tracked-file runtime payload.
