# UEEF v1.4.0

UEEF 1.4.0 adds Smart Environment Bootstrap. Before engineering begins, UEEF selects task-relevant environment profiles, verifies tools, skills, MCPs, runtime, and validators, and emits evidence-backed readiness.

## Highlights

- Pack 50 profiles: Core, Frontend, Backend, Database, UIUX, DevOps, AI, and Optional.
- Core and AI are mandatory; other profiles are selected from task and repository evidence.
- Mandatory missing blocks, Recommended missing warns with an exact action, and Optional missing never blocks.
- PowerShell and Unix bootstrap checkers support current path, version, skill, runtime, and tool detection without exposing secrets.
- Bootstrap is required before project inspection, architecture detection, planning, implementation, and quality gates.
- Added environment gate, readiness scorecard, checklist, runtime output contract, and validation requirements.

## Upgrade

```powershell
git pull
powershell -ExecutionPolicy Bypass -File .\scripts\validate-framework.ps1
powershell -ExecutionPolicy Bypass -File .\scripts\environment-bootstrap.ps1
powershell -ExecutionPolicy Bypass -File .\scripts\install-codex.ps1
```

Select only profiles required by the current task. A missing optional tool is not a failure; a missing selected Mandatory dependency is a blocker until safely repaired.
