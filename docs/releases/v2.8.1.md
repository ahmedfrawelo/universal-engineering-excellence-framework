# UEEF 2.8.1

Released: 2026-07-12

## Summary

This patch closes the gap that allowed a code-changing task to claim UEEF compliance without running a child agent or recording a routing reason.

## Changes

- Any code change is classified at least T1.
- T1 through T4 code changes emit `spawnAgents: true` whenever agent tooling is available.
- Route schema 3 adds `codeChange`, `noSpawnReason`, and `routeEvidenceRequired` evidence.
- A visible route line is required before the first project command or edit.
- `TOOL_UNAVAILABLE` is the only valid no-spawn reason for code-changing tasks.
- Runtime status validates the installed loader, global instructions, routing gate, route schema, and active-state contract.
