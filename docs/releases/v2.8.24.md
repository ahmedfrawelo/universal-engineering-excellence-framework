# UEEF 2.8.24

Release date: 2026-07-16

## Summary

UEEF 2.8.24 adds a Chrome control readiness contract so browser verification failures are repaired and classified correctly before any task can report `BLOCKED`.

## Changes

- Added `framework/51-browser-session-control/15-chrome-control-readiness.md`.
- Required supported Chrome skill bootstrap, exact `user.openTabs()` object selection, `claimTab()`, automatic stale-ownership repair, `VERIFIED_HANDOFF`, and `chrome.tabs.finalize(...)`.
- Rejected browser-verification `BLOCKED` lifecycle transitions unless `CHROME_EXTERNALLY_UNAVAILABLE` is independently proven.
- Clarified the recovery state machine: one readiness/self-repair pass, then current handoff, with no visible retry loop and no user restart request from task-local bridge failures.
- Added PowerShell and Unix contract coverage for the readiness module and runtime sequence fields.
