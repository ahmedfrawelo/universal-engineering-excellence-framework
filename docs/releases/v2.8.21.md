# UEEF 2.8.21

Release date: 2026-07-16

## Summary

UEEF 2.8.21 adds a response-readability rule for Arabic and other RTL prose that mixes English identifiers.

## Changes

- Mixed Arabic/English prose may use display-only bidirectional isolation when it improves readability.
- Hidden direction characters remain prohibited in code blocks, terminal commands, copyable file paths, JSON/YAML, source files, configuration, and saved artifacts.
- The generated Codex loader and AGENTS instructions now include the same rule.
- Validation now checks that the final-response format and runtime generator preserve this policy.

## Validation

- Run `scripts/validate-framework.ps1`.
- Run `scripts/validate-framework.sh`.
- Run `scripts/ueef-audit.ps1 -Quick`.
