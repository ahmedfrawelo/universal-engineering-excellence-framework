# UEEF 2.13.0

Release date: 2026-07-23

This release adds a governed set of optional workflow utilities. They are read-only or opt-in by default and do not use process or network probes to claim that a configured MCP is callable.

## Task and workflow evidence

- `get-ueef-task-preflight.ps1` composes routing, capability selection, workflow decisions, and conditional health evidence without starting MCPs or claiming configured capabilities are callable.
- Specification workflows now include `clarifications.md` and `convergence.md`; the validator checks the corresponding readiness contracts.
- `get-diff-impact.ps1` and `get-diff-impact.sh` provide bounded, path-based impact suggestions. Their confidence is explicitly `HEURISTIC`, and callers must still inspect imports, contracts, and tests.
- `get-task-budget-advice.ps1` and `get-task-budget-advice.sh` provide a heuristic time budget only; they are not model-token telemetry or a cost quote.

## Local governance utilities

- Project memory is opt-in, project-local, retention-bounded, and rejects credential-like values.
- Team-policy profiles resolve declared team guidance without automatically changing policy.
- Evidence-pack export records runtime, health, and optional diff evidence while redacting credential-like values.
- Learning-loop records require human approval; workspace-boundary discovery and safe-refactor recipes are read-only/scaffolded helpers; onboarding is a dry-run preview.

## Compatibility and platform support

- The executable adapter matrix supports and tests only Codex, Cursor, and a generic AGENTS-compatible target. It does not imply installers for other assistants.
- The optional tools above are Windows-first unless a matching `.sh` command is present. Unix support in this release is limited to `get-diff-impact.sh` and `get-task-budget-advice.sh`.
