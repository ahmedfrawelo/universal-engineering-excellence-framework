# UEEF v2.7.1

This hardening release follows a full review of agent orchestration and the complete framework runtime.

## Agent Orchestration

- Critical risk score `3` now requires an explicit risk floor.
- Parallel routing requires at least two independently owned workstreams.
- Routing records agent and named-model availability and falls back safely when unavailable.
- PowerShell and Unix selectors share a versioned JSON schema and behavioral tests.

## Runtime and Installation

- Agent names and runtime targets are constrained before any write or replacement.
- Active-state metadata is agent-aware and status checks validate its contents.
- Runtime drift checks cover the full copied source plus generated-loader contracts.
- Generated runtime updates return to the recorded Git source and resync the installation.
- Windows and Unix installers support explicit non-interactive replacement and preserve current loader behavior.

## Assurance

- Quality-gate selection emits only existing framework paths and has representative tests.
- Unix audit now checks source hygiene, shell syntax, release parity, and runtime path controls.
- Unix bootstrap/status behavior is aligned with Windows contracts.
- Documentation, code fences, security reporting, release history, and SemVer guidance were corrected.
