# UEEF 2.8.23

Release date: 2026-07-16

## Summary

UEEF 2.8.23 prevents pending screenshot evidence or task-local Chrome-control degradation from creating false `BLOCKED` task endings after implementation and non-visual validation can continue or has passed.

## Changes

- Pending screenshot evidence and pCloud screenshot delays are no longer valid `BLOCKED` reasons by themselves.
- Task-local Chrome-control degradation remains an active verification gap, not proof that Chrome is externally unavailable.
- Required visual verification still blocks `COMPLETE` claims unless the current user-owned tab is verified or a current trusted handoff covers the same code state.
- Goal lifecycle validators now reject `BLOCKED` transitions marked only by pending screenshot evidence.
