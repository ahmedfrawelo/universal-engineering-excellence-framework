## Summary

This patch release completes a full repository hardening pass across documentation integrity, generated indexes, workspace cleanup, installation, runtime synchronization, release inventory, and Windows/Unix verification parity.

## Changes

- Generated every pack index and the framework master index from the real file tree, with exact drift tests.
- Corrected obsolete selector and example paths and added validation for both Markdown links and plain repository-relative paths.

- Restricted runtime payloads to governed release files and rejected environment, credential, key, and certificate material.
- Added source and staged validation, atomic runtime switching, rollback, stale-root removal, and managed preservation of user-owned `AGENTS.md` content.
- Replaced shell JSON text parsing and construction with structured Node helpers and added Unix runtime drift enforcement.
- Pinned the design-engineering skill source to audited commit `6bf24434f7730ad169077756cf9c7cd7bd675fc6`.
- Added exact Markdown inventory enforcement in addition to the existing minimum threshold.

The release remains backward compatible. Existing Codex-managed `AGENTS.md` files are migrated to a managed block; unrelated user-authored content is retained.
