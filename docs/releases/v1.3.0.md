# UEEF v1.3.0

UEEF 1.3.0 adds a permanent Engineering Guardian layer. Every non-trivial task now establishes a baseline, selects regression monitors, checks affected quality dimensions, performs self-criticism, and passes a final guardian gate.

## Highlights

- Pack 49 with zero-regression policy, breaking-change prevention, production, UI, performance, security, architecture, design, dependency, database, API, debt, health, complexity, consistency, folder, improvement, self-criticism, final-gate, health-score, maintenance, future-proof, product-review, and final-checklist modules.
- High and Critical security findings, critical performance regressions, broken contracts, accessibility regressions, and known unowned degradation are release blockers.
- Nearby safe improvements are required when discovered during a scoped change, while unrelated refactoring remains out of scope.
- Core, Master Loader, Runtime Sequence, AI review, quality gates, scorecards, checklists, validation, and release metadata now reference the Guardian layer.

## Upgrade

```powershell
git pull
powershell -ExecutionPolicy Bypass -File .\scripts\validate-framework.ps1
powershell -ExecutionPolicy Bypass -File .\scripts\install-codex.ps1
```

For every task, record the affected baseline, selected monitors, self-criticism, final gate, and residual risk. The Guardian complements packs 45-48 and does not replace their specialized contracts.
