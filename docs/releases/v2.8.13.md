# UEEF 2.8.13

## Non-Destructive Runtime Updates

Runtime synchronization now updates the existing runtime in place instead of deleting and recreating it. This preserves files already imported by active task kernels, including the Node REPL browser client used to control an existing user-owned Chrome tab.

## Regression Coverage

The runtime hardening test now creates an active-task sentinel, runs another sync, and requires the sentinel to survive. It also rejects the previous destructive runtime removal command.
