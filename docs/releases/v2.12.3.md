# UEEF 2.12.3

Release date: 2026-07-23

## Unified health evidence

`scripts/get-ueef-health.ps1` combines the structured runtime status with the declared and observed capability inventory. It is read-only: it does not install software, launch MCP processes, or send network probes.

The report exposes schema version, runtime state, capability health counts, detailed items, and actionable diagnostics. A required missing or disabled capability, or an inactive runtime, produces `FAIL`; optional gaps produce `DEGRADED`; configured-but-unverified capabilities remain usable and are not represented as callable.

`scripts/ueef-status.ps1 -Json` now provides a stable machine-readable status contract while retaining its existing human-readable output. `scripts/test-ueef-health.ps1` validates the active runtime end to end after installation.

The runtime release policy also excludes the framework's documented transient `.tmp`, `.log`, build, and local-workspace paths when Git metadata is unavailable, preserving the same tracked-file behavior in isolated test and installer environments.
