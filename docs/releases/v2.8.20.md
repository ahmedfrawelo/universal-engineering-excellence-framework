# UEEF 2.8.20

Release date: 2026-07-16

## Summary

UEEF 2.8.20 hardens runtime synchronization and release validation while making routine audits easier to run frequently.

## Changes

- Runtime sync still updates in place, but now prunes stale files from owned UEEF folders: `framework`, `scripts`, `docs`, `examples`, `tools`, and `assets`.
- Task-local root files in an active runtime remain preserved during sync.
- Runtime drift checks now detect extra stale files in owned runtime folders.
- `validate-framework.ps1` validates manifest-controlled markdown minimums, entrypoints, assets, release notes, and release date format.
- `ueef-audit.ps1` records per-check durations and supports `-Quick` for routine assurance without the full runtime-hardening scenario.
- Runtime-hardening tests cover stale owned-file pruning and root sentinel preservation.

## Validation

- Run `scripts/test-runtime-hardening.ps1`.
- Run `scripts/validate-framework.ps1`.
- Run `scripts/validate-framework.sh`.
- Run `scripts/ueef-audit.ps1`.
