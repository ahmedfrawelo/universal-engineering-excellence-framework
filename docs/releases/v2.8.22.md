# UEEF 2.8.22

Release date: 2026-07-16

## Summary

UEEF 2.8.22 makes mixed-direction Arabic/English response isolation mandatory instead of optional.

## Changes

- Every inline English word, identifier, product name, or short LTR phrase in Arabic/RTL prose must be isolated for display readability.
- Validation rejects optional wording such as "may be isolated" or "when it improves readability" in the mixed-direction response policy.
- Hidden direction characters remain prohibited in code blocks, terminal commands, copyable file paths, JSON/YAML, source files, configuration, and saved artifacts.

## Validation

- Run `scripts/validate-framework.ps1`.
- Run `scripts/validate-framework.sh`.
- Run `scripts/ueef-audit.ps1 -Quick`.
