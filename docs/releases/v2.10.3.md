# UEEF 2.10.3

Release date: 2026-07-18

## Summary

This patch release closes the independent security and transactional-integrity findings from the 2.10.2 repository audit.

## Changes

- Made Git-tracked files the single release inventory on both PowerShell and Node paths; untracked files under owned folders no longer enter a runtime payload.
- Rejects every symlink or reparse-point segment between the source root and a release file, including linked parent directories.
- Centralized PowerShell runtime drift comparison so status and standalone drift checks use the exact same inventory and hash rules.
- Records the generated loader SHA-256 in active state and validates it on Windows and Unix, so any loader mutation invalidates activation.
- Extended runtime synchronization rollback to restore the previous `UEEF-ACTIVE.json` and `AGENTS.md` content after post-activation failure.
- Extended Windows and Unix installer rollback to restore the previous active state and remove state created by failed first installs.
- Added regression coverage for tracked-file parity, linked-parent rejection, rollback after state writes, first-install cleanup, and Windows filesystem-root normalization.

## Compatibility

The release is backward compatible. Runtime payloads become stricter: untracked content and files reached through links are intentionally excluded or rejected.
