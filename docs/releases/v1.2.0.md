# UEEF v1.2.0

UEEF 1.2.0 adds a governed design system layer that makes reuse, visual language, tokens, components, patterns, and interaction behavior enforceable across projects.

## Highlights

- Pack 48 with 31 modules for design governance, language, tokens, color, typography, icons, spacing, sizing, radius, borders, shadows, elevation, motion, components, registry, patterns, layouts, templates, themes, responsive behavior, interactions, overlays, drift, reuse, quality, scorecards, and review.
- Mandatory search order: project, design system, component registry, shared components, shared services, pattern library.
- Reuse before extension, extension before generalization, generalization before new creation.
- Token enforcement for all visual values and explicit exceptions with owners and expiry.
- One icon family, approved typography roles, shared radius, border, shadow, elevation, spacing, sizing, and motion systems.
- Governance gate, scorecard, checklist, and review template integrated with packs 46 and 47.

## Upgrade

Pull the release, validate, and update the agent runtime. For UI work, select pack 48 with pack 46 reuse governance and pack 47 theme, responsive, accessibility, interaction, security, and performance contracts.

```powershell
git pull
powershell -ExecutionPolicy Bypass -File .\scripts\validate-framework.ps1
powershell -ExecutionPolicy Bypass -File .\scripts\install-codex.ps1
```

Existing product identity is preserved. Pack 48 governs how the existing system is found, extended, and reused; it does not authorize replacing a valid product design without evidence.
