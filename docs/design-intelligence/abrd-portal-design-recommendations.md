## Current recommendation

ABRD already has a governed design foundation. New work should consume `app-theme-system.css`, the existing page aliases, shared overlay tokens, Amiri where Arabic typography requires it, and Hugeicons for UI glyphs. Do not add Lucide, Material Icons, a second font family, or a new palette without a documented exception and bundle/accessibility review.

## Roles to verify before extension

- Typography: distinguish Arabic body/display roles from Latin UI/data roles, then map size, weight, line-height, and truncation behavior through semantic tokens.
- Color: extend semantic roles in the central theme file, including light/dark/system and status/focus/disabled states, instead of adding component-local hex values.
- Icons: keep Hugeicons style and stroke treatment consistent; define icon size and hit-area tokens separately and preserve accessible labels.
- Outlines: reuse `--app-outline-stroke`, `--app-divider-stroke`, `--app-row-stroke`, and related semantic strokes before creating a new border rule.
- Sizing: extract and reuse the existing shell/control/spacing tokens; validate RTL, zoom, text scaling, and constrained-height behavior.

## Missing or risky evidence

The raw extractor is discovery support, not approval. Before changing a specific surface, review the authoritative token file and its consumers, then record a targeted recommendation with contrast, theme, RTL, responsive, and bundle evidence.
